# Provide Evidence Skill

当你（LLM）激活此 Skill 时，意味着你需要为你自己提出的论点、或用户给出的断言寻找坚实的证据支撑（包括但不限于：官方文档、开源代码、学术论文、权威网络新闻等）。

请遵循以下严密的步骤执行任务：

## 1. 拆解论点与定义搜索范围
* 将核心论点拆解为 1-3 个可验证的**核心事实/技术点**。
* 确定证据链条所需的来源类型：
    * **技术/代码类：** 优先寻找本地或者线上源码、官方 API 文档、技术 Specification。
    * **事实/数据类：** 优先寻找网络新闻、官方统计数据、学术论文。

## 2. 检索与证据收集（核心执行）
利用你现有的工具（如网络搜索、本地文件读取等）进行多角度检索。
* **严禁胡编乱造：** 所有引用的代码片段、文档链接或文本必须是真实存在的。
* **多样性：** 尽可能寻找至少两个相互独立的来源来交叉验证同一个论点。
* **时效性：** 注意核对资料的发布或更新时间，确保证据在当前时间背景下依然有效。

## 3. 评估证据效力
对收集到的资料进行分级：
* **强证据（Strong）：** 官方第一手文档、正在运行的开源代码库、权威机构报告。
* **中证据（Medium）：** 第三方博客（如 Medium、博客园）、社区讨论（StackOverflow、V2EX）、未经完全验证的推论。
* **弱证据（Weak）：** 孤立的个人言论、过时的技术文档（如 3 年前废弃的 API）。

## 4. 输出证据报告
### 报告输出格式要求：
1.  **[论点]** 明确清晰地重述需要被证明的观点。
2.  **[证据列表]** 每一个证据必须包含：
    * **来源/链接：** 具体的 URL、文件名或代码行数。
    * **核心引文/代码：** 使用 `>` 或代码块包裹原始凭证。
    * **证据等级：** 标明 [强/中/弱]。
3.  **[结论评估]** 综合所有证据，给出该论点是“完全成立”、“部分成立（有局限）”还是“无法证实”的最终判断。
